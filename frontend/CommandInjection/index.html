Command Injection

In this example, the server allows users to echo something. You can try to input something like: hey you The important aspect of this, is that the backend server is executing a system command. The backend is taking user input and appending it to the command, this is very bad. exec("echo " + userinput, (e, stdout, stderr)) One feature of system commands is that you can string multiple commands together using various symbols. Try these (on windows try dir instead of ls): yo && ls yo && whoami Bad right? If you must allow user input driven system commands (but why?), carefully validate user input using a whitelist. Additionally, the user running the server app should have very limited privileges.
